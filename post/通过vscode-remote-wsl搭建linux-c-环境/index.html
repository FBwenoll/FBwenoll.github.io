通过VsCode Remote WSL搭建linux C 环境

Jan 12, 2023
1 min read

安装编译工具链

  • 安装编译工具
    sudo apt update
    sudo apt-get install build-essential gbd
    sudo apt-get install cmake
  • 验证是否安装成功
    gcc --version
    g++ --version
    gbd --version
    cmake--version

安装Vscode插件

  • 首先通过remote_wsl连接WSL2
  • 在连接WSL的状态下安装C++、Cmake、CmakeTools等一系列工具

创建工程

  • 创建如下目录 
     Projects
 └── Demo
     ├── include
     │   └── HelloWorld.h
     ├── main.cpp
     └── src
         └── HelloWorld.cpp
 其中Projects为工作空间，Demo为项目
  • F5选择工具链即可进行编译运行,但此时会报错，因为此时我们没有将新建的include&src目录添至includePath中去。

添加includePath

  • 通过ctrl+shift+p打开命令窗口，并输入configuration，即可在.vscode文件夹中看到c_cpp_properties.json以及task.json文件。
  • 在c_cpp_properties.json中includePath内添加工程内头文件所在的文件目录： 
     {
     "configurations": [
         {
             "name": "Linux",
             "includePath": [
                 "${workspaceFolder}/**",
                 "${workspaceFolder}/Demo/include"//头文件目录
             ]
         ...
     }
 ],
 "version":4
 }
  • 在task.json文件中args内添加需要编译的cpp文件目录： 
        "args": [
    "-fdiagnostics-color=always",
    "-g",
    "${workspaceFolder}/Demo/**.cpp",//编译Demo目录下所有.cpp文件
    "${workspaceFolder}/Demo/src/**.cpp",//编译Demo/src目录下所有.cpp文件
    "-o",
    "${fileDirname}/${fileBasenameNoExtension}",
    //添加如下内容以启用多文件编译
    "-I",
    "${workspaceFolder}/Demo/include",//头文件目录
    "-pthread"
],
